# 📘 คู่มือการใช้งานระบบ AI Hybrid Trading (v6.1)

คู่มือนี้สำหรับผู้ใช้งาน **SelfLearningEA** เวอร์ชั่น 6.1 ขึ้นไป ซึ่งเน้นความทนทาน (Resilience) และความแม่นยำด้วยระบบ Hybrid AI

---

## 1. การติดตั้ง AI สำรอง (Ollama Setup)
เป็นฟีเจอร์ใหม่ใน v6.1 เพื่อแก้ปัญหา Gemini API โดนจำกัดโควตา (Rate Limit)

1. **ดาวน์โหลด Ollama**: ไปที่ [ollama.com](https://ollama.com) และติดตั้งลงเครื่อง
2. **โหลดโมเดล**: เปิด Terminal/Command Prompt แล้วรันคำสั่ง:
   ```bash
   ollama run llama3.1
   ```
3. **การทำงาน**: เมื่อติดตั้งแล้ว บอทจะตรวจหา Ollama อัตโนมัติ หาก Gemini ติดปัญหา บอทจะสลับไปเรียก Llama 3.1 ในเครื่องคุณมาวิเคราะห์แทนทันที

---

## 2. กลยุทธ์การเทรด (Trading Core)
บอทใช้ 2 กลยุทธ์หลักจากแนวคิด ICT/SMC:

### ก. ICT Consolidation (7-Step):
- ตรวจสอบโซนราคาที่ไม่มีทิศทางชัดเจน
- รอการทำ Liquidity Sweep (กวาดราคาออกไปกิน SL)
- เข้าเทรดเมื่อราคากลับเข้าสู่โซนดั้งเดิมในระนาบ M15

### ข. Trend OTE (Optimal Trade Entry):
- ระบุทิศทางขาขึ้น/ลงที่รุนแรง
- รอราคาพักตัวเข้าสู่โซน Fibonacci 62% - 79% (OTE Zone)
- ใช้เป้าหมายกำไรที่ SD 2.0 และ 2.5 ของกรอบราคาเดิม

## 3. การตั้งค่าที่สำคัญ (config.py)

| ค่าพารามิเตอร์ | คำอธิบาย |
| :--- | :--- |
| `AI_MODEL` | ชื่อรุ่น AI หลัก (แนะนำ: `gemini-2.0-flash-lite`) |
| `AI_CONFIDENCE_THRESHOLD` | ความมั่นใจขั้นต่ำ (%) ที่จะให้บอทเปิดออเดอร์ (70%) |
| `AI_CHECK_INTERVAL` | ความถี่ในการถาม AI (แนะนำ 7200 วินาที หรือ 2 ชม. เพื่อถนอม Quota) |
| `RECOVERY_TRIGGER_PERCENT` | ระดับ Drawdown ที่จะเปิดโหมด Safety Brake (-10%) |
| `ENABLE_OLLAMA_FALLBACK` | เปิดใช้งาน AI สำรองในเครื่อง (ต้องตั้งเป็น `True`) |

## 4. ระบบจัดการความเสี่ยง (Risk & Safety)

### 🚨 Global Recovery Mode (v6.1)
- **Safety Brake**: เมื่อพอร์ตติดลบเกินค่าที่กำหนด บอทจะหยุดส่งออเดอร์ใหม่ทันที
- **Non-Closing**: บอทจะไม่ปิดไม้ของคุณอัตโนมัติ เพื่อให้คุณบริหารจัดการหรือรอราคากลับมาเอง
- **Auto-Reset**: ระบบจะปลดล็อคเบรกมือให้เองเมื่อพอร์ตกลับมาอยู่ในระดับปลอดภัย (DD ดีกว่า -5%)

### 🛡️ Virtual Stop Loss
- จุดตัดขาดทุนจะคำนวณและเก็บไว้ในหน่วยความจำของบอทเท่านั้น ไม่มีการส่งไปที่ Server ของโบรกเกอร์ เพื่อป้องกันการโดนลากไปกิน SL จากความผันผวนชั่วคราว

## 5. การตรวจสอบและบำรุงรักษา

- **Terminal Logs**: จะแสดงสถานะการวิเคราะห์ AI อย่างละเอียด (หากสลับไปใช้ Llama จะขึ้นไอคอน 💎)
- **Database**: ข้อมูลการเทรดทั้งหมดถูกเก็บใน `trades.db` สามารถใช้ `analyze_trades.py` เพื่อดู Report ย้อนหลังได้
- **Session Check**: บอทจะกรองช่วงเวลาสำคัญ เช่น ข่าวแรงหรือช่วงปิดตลาด เพื่อลดความเสี่ยง
